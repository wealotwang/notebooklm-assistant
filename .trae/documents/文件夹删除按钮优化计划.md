# 文件夹删除按钮优化计划

您希望删除文件夹的操作更直观、更快捷，而不需要通过右键菜单。

## 1. 需求分析
*   **显式删除按钮**: 在文件夹胶囊（Chip）内部添加一个“叉号” (x) 按钮。
*   **按需显示**:
    *   只有当文件夹被**选中**（激活）时，才显示叉号。
    *   或者（可选方案）：当鼠标**悬停**在文件夹上时，显示叉号。
    *   您提供的参考图似乎是“选中状态下显示红色叉号”，这符合常见的 Tag/Chip 设计模式。
*   **点击逻辑**: 点击叉号直接触发删除确认，点击胶囊其他区域则是切换文件夹。

## 2. 设计方案

### A. UI 结构调整 (`content.js`)
*   修改 `renderFolders` 函数。
*   在生成 `li` 元素时，如果是自定义文件夹（非 `all`），且处于激活状态（或者我们采用悬停显示），则 append 一个 `<span class="nlm-folder-remove">×</span>`。

### B. CSS 样式增强 (`styles.css`)
*   `.nlm-folder-remove`:
    *   定位：放在胶囊右侧。
    *   样式：小巧的叉号，鼠标悬停变红。
    *   交互：`opacity` 控制显示/隐藏，或者仅在 `.active` 状态下 `display: block`。

### C. 交互逻辑 (`content.js`)
*   为叉号绑定 `click` 事件。
*   **关键点**: 需要 `e.stopPropagation()` 阻止冒泡，防止点击删除时误触发文件夹切换。
*   删除逻辑复用之前的 `deleteFolder`，并保留确认弹窗。

## 3. 实施步骤
1.  **修改 `content.js`**: 在 `renderFolders` 中，为非系统文件夹添加删除按钮 DOM 结构。
2.  **修改 `styles.css`**: 添加删除按钮的样式（默认隐藏，选中/悬停时显示）。
3.  **版本更新**: 升级至 `v2.1.22`。
