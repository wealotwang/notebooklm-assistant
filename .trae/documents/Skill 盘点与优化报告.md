# Skill 体系最终优化方案

您的思路非常清晰。关于“版本号记录”的归属问题，我的分析如下：

## 1. 核心回答：版本号记录属于 Skill，而非 Todo
*   **不要放 Todo**：`todo.json` 应该只关注**“业务目标”**（如“修复 Bug A”、“开发功能 B”）。
*   **放入 Skill**：**“记录版本号”** 属于**“流程动作”**。这正是 Skill 存在的意义——把这些重复的、琐碎的流程自动化。如果您每次都要在 Todo 里写一条“记录版本号”，那就是在给您自己增加负担。

## 2. 最终优化方案 (The Master Plan)

我将基于您的所有反馈（去重、不打包、强化版本控制），把 `收尾工作` Skill 重构为项目的**“智能管家”**。

### 重构后的 `收尾工作` Skill 逻辑：

1.  **自动同步进度 (Auto-Sync)**
    *   **动作**：读取 `todo.json` 中今日完成的任务。
    *   **价值**：解决 Overlap，直接用 Todo 状态生成日报，不再需要维护 `task_plan.md`。

2.  **版本控制自动化 (Version Control)**
    *   **动作**：
        *   检测是否有代码变更。
        *   **询问/确认**：是否需要升级 `manifest.json` 版本号？
        *   **写入**：如果有升级，自动在 `CHANGELOG.md` 增加一条记录。
        *   **提交**：执行 `git commit`，将版本号和完成的任务作为 Commit Message。

3.  **文档归档 (Documentation)**
    *   **动作**：将今日复盘写入 `dev_docs/Project_Review_Timeline.md`（修正路径）。

4.  **清理 (Cleanup)**
    *   **动作**：不做打包。只检查根目录是否有临时文件需要清理。

## 3. 执行步骤

1.  **更新 Skill**：立即修改 `.trae/skills/收尾工作/SKILL.md`，写入上述新逻辑。
2.  **清理文档**：将根目录下旧的 `task_plan.md` 等文件归档/删除，确保 `todo.json` 是唯一真理。
