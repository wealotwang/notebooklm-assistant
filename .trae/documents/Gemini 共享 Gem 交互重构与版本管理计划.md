# Gemini 共享 Gem 交互重构与自动化计划

为了提供更自然、更像原生的使用体验，我将对“共享 Gem”功能进行全面的交互重构，确保它在侧边栏中既美观又易于管理。

## 1. 交互重构：自动固定与原生化菜单

### A. 自动固定 (Auto-Pin)
*   **无感收藏**: 取消侧边栏的 `+` 按钮。
*   **实现**: 当您访问任何共享 Gem 页面 (`/gem/...`) 时，插件将自动从页面 DOM（识别您提供的 `bot-name-container` 结构）中精准提取 Gem 的真实名称。
*   **静默保存**: 系统会自动将该 Gem 的 ID、名称和 URL 保存到您的侧边栏，实现“看过即留下”。

### B. 原生化列表展示 (Enhanced UI)
*   **显示真实名称**: 修复目前只显示 "Gemini" 的问题，侧边栏将完整展示 Gem 的具体名称（如“医药行业资料补充研究”）。
*   **三点菜单 (Native-like)**: 
    *   将之前的 `×` 按钮升级为标准的“三个点”更多操作按钮。
    *   点击“三个点”将弹出一个原生风格的小菜单，提供“移除此共享 Gem”选项。
*   **视觉适配**: 列表项将采用与 Gemini 原生侧边栏一致的样式，包括悬停高亮和图标间距。

## 2. 版本管理与“还原点”创建 (Rollback Strategy)

为了确保您可以随时安全地回溯版本，我将执行以下流程：

1.  **版本升级**: 将 `manifest.json` 版本号升级至 `3.0.0.19`。
2.  **Git 详细提交**: 提交所有变更，并记录详细的功能日志。
3.  **创建还原点 (Git Tag)**: 
    *   我会创建一个名为 `v3.0.0.19` 的 Git 标签。
    *   **价值**: 这就像游戏的存档点，如果未来您觉得这个新交互不如预期，我们可以通过这个标签一键回滚到当前或之前的任何状态。
4.  **GitHub 同步**: 将代码和标签同步推送到 GitHub 仓库。
