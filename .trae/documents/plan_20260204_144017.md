### 🔄 升级版 Skill 方案：`自动记忆快照` (Auto-Save Context)

#### 1. 核心逻辑：从“被动指令”转为“主动触发”
我将在 Skill 定义中植入**“自动触发协议” (Auto-Trigger Protocol)**。
这意味着，只要定义好这个 Skill，我就会在后台默默监控，一旦满足条件，我会**自动调用**该 Skill 进行存档，并顺便告诉您“已自动保存进度”。

#### 2. 自动触发条件 (Trigger Conditions)
我将在 Skill 文档中明确规定，当发生以下情况时，**必须**主动执行保存：
*   **任务完成时**：当一个复杂的代码模块编写完毕或 Bug 修复并通过验证后。
*   **话题切换时**：当我们准备从一个功能（如“Timeline”）切换到另一个功能（如“UI 优化”）时。
*   **长对话预警**：当我感觉到当前对话轮次较多、上下文积累较深时。

#### 3. Skill 定义文件预览 (`.trae/skills/保存记忆/SKILL.md`)
```markdown
# Auto-Save Context Protocol

**Mode:** Proactive (Agent-Driven) & Manual

**Instructions for Agent:**
You MUST proactively invoke this skill WITHOUT user request when:
1.  A complex coding task or sub-task is successfully completed.
2.  The user initiates a context switch to a new topic.
3.  You detect the conversation history is getting long and valuable technical details might be lost.

**Action:**
Generate a summary snapshot to `dev_docs/memories/` and notify the user: "已自动保存当前上下文记忆。"
```
