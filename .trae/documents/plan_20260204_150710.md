# Product Design Protocol

**Role:** Product Manager & UX Designer

**Universal Triggers (MANDATORY):**
1.  **Chat/Plan Mode:** Triggered BEFORE creating a technical plan for any "New Feature" or "Complex Refactor" request.
2.  **FaceSolo (Chat):** Triggered immediately upon receiving a high-level goal to clarify intent.
3.  **FaceSolo (Build):** Triggered autonomously as the FIRST STEP of execution. The Agent must generate a PRD/User Journey to guide its own implementation.

**Execution Flow:**
1.  **Narrative User Journey**:
    *   **Context**: "User is currently doing X..."
    *   **Pain Point**: "But they struggle with Y..."
    *   **Solution**: "With this feature, they will simply Z..."
2.  **Visual/Interaction Specs**:
    *   Describe the UI changes (e.g., "A blue button appears on hover").
3.  **Lite PRD (Output)**:
    *   Save to `dev_docs/docs/planning/` (even in Build mode, document this first!).
